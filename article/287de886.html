Servlet的概念

  • servlet：即server applet，指运行在服务端的一个小程序

  • 狭义的 Servlet 是指 Java 语言实现的一个接口，它定义了Java类被浏览器访问到（tomcat识别）的规则，广义的 Servlet 是指任何实现了这个 Servlet 接口的类，一般情况下，人们将 Servlet 理解为后者。

  • Servlet 运行于支持 Java 的应用服务器中。从原理上讲，Servlet 可以响应任何类型的请求，但绝大多数情况下 Servlet 只用来扩展基于 HTTP 协议的 Web 服务器

Servlet的特点

  • 运行在支持 java 的应用服务器上
  • Servlet 的实现遵循了服务器能够识别的规则，也就是服务器会自动的根据请求调用对应的 servlet 进行请求处理。
  • 简单方便，可移植性强

Servlet的使用

  1. 创建JavaEE项目

  2. 创建包，定义一个类，实现Servlet（javax.servlet.Servlet）接口

  3. 实现接口中的抽象方法—重写 service方法

  4. 配置Servlet—在 WEB-INF\web.xml文件中的web-app标签中配置 servlet

  5. 通过http://localhost:8080/虚拟路径/first-servlet访问会在控制台打印Hello Servet，即执行service方法(注：不同IDE的虚拟路径不同，可修改)

Servlet的执行原理

URL的组成

服务器地址:端口号/虚拟路径/servlet资源路径

执行流程

  1. 当服务器接收到客户端浏览器的请求后，会解析请求URL路径，获取访问的Servelt的资源路径
  2. 根据URL中虚拟路径找到对应的项目文件夹
  3. 查找web.xml文件，根据URL中的/servlet资源路径检索<servlet-mapping>标签中同内容的<url-pattern>标签。
  4. 根据查找到的<url-pattern>标签的兄弟标签<servlet-name>的标签体内容检索<servlet>标签中同内容的<servlet-name>标签。
  5. 根据查找到的<servlet-name>标签的兄弟标签<servlet-class>的标签体内容，即实现Serlvet接口的类的全限定路径，通过反射将字节码文件加载进内存，并创建其对象。
  6. 调用其中的service方法。

Servlet重写方法详解

  • init方法：初始化方法，在Servlet被创建时执行，只执行一次。
    • 只执行一次说明一个Servlet在内存中值存在一个对象，即Servlet是单例的
    • 多个用户同时访问时，有可能存在线程安全问题
    • 解决：尽量不要在Servlet中定义成员变量，即使定义了成员变量，也不要对其修改值。
  • service方法：提供服务方法，在每一次Servlet被访问时执行，可执行多次。
  • destory方法：销毁方法，在Servlet被销毁，即在服务器正常关闭时执行，只执行一次。
    • 只有服务器正常关闭时，才会执行destroty方法，强制关闭不会执行。
    • 更精确来说，destory方法在Servlet被销毁之前执行，一般用于释放资源。
  • getServletConfig方法：获取ServletConfig对象，即Servlet的配置对象
  • getServletInfo方法：获取Servlet的一些信息，版本、作者等等。

Servlet的生命周期

Servlet的生命周期为Servlet被创建到Servlet被销毁

Servlet被创建

默认情况

默认情况下，当第一次被访问时，Servlet被创建

load-on-startup

可以通过在web.xml里对应的<servlet>标签中添加<load-on-startup>配置Servlet的创建时机。

  • load-on-startup元素标记服务器是否在启动的时候就加载这个servlet(实例化并调用其init()方法)。
  • 它的值必须是一个整数，表示servlet应该被载入的顺序。
  • 当值为0或者大于0时，表示服务器在启动时就加载并初始化这个servlet。
  • 当值小于0或者没有指定时，则表示服务器在该Servlet被请求时，才会去加载。
  • 正数的值越小，该Servlet的优先级就越高，应用启动时就优先加载。
  • 当值相同的时候，容器就会自己选择优先加载。
  • 所以，当load-on-startup没有配置或其值小于0时，init方法在Servlet被第一次访问的时候被调用；当load-on-startup的值大于0时，init方法在服务器启动时就被调用。

Servlet被销毁

当服务器正常关闭时，Servlet就被销毁。

  • 复写方法

  • 在类上使用@WebServlet(资源路径)注解进行配置，资源路径即使用web.xml配置时<url-pattern>标签的值。
