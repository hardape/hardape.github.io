浏览器现状

PC端常见浏览器

360浏览器、谷歌浏览器、火狐浏览器、QQ浏览器、百度浏览器、搜狗浏览器、IE浏览器

移动端常见浏览器

UC浏览器、QQ浏览器、欧朋浏览器、百度手机浏览器、360安全浏览器、谷歌浏览器、搜狗手机浏览器、猎豹浏览器…

注意

国内的UC和QQ、百度等手机浏览器都是根据Webkit修改过来的内核.

总结

兼容移动端主流浏览器，处理Webkit内核浏览器即可.

手机屏幕现状

  1. 移动端设备屏幕尺寸众多，碎片化严重
  2. Android设备有多种分辨率：480×800，480×854，540×960，720×1280，1080×1920等，还有2k，4k屏.
  3. 今年来iPhone的碎片化也加剧了，其设备的主要分辨率有：640×960，640×1136，750×1334，1242×2208等.
  4. 作为开发者无须关注这些分辨率，因为我们常用的尺寸单位是px.

视口

viewport

视口是浏览器显示页面内容的屏幕区域。 视口可以分为布局视口、视觉视口和理想视口。

布局视口

layout viewport

  1. 一般移动设备的浏览器都默认设置了一个布局视口，用于解决早期的PC端页面在手机上显示的问题。
  2. iOS, Android基本都将这个视口分辨率设置为980px，所以PC上的网页大多数都能在手机上呈现，只不过元素看上去很小，一般默认可以通过手动缩放页面，用户体验度差。

视觉视口

visual viewport

  1. 视觉视口是指用户正在看到的网站的区域。
  2. 用户可以通过缩放去操作视觉视口，但不会影响布局视口，布局视口仍然保持原来的宽度。

理想视口

ideal viewport

  1. 理想视口对设备来讲，是最理想的视口尺寸
  2. 是为了是网站在移动端有最理想的浏览和阅读宽度而设定
  3. 需要手动添加 meta视口标签通知浏览器操作
  4. meta视口标签的主要目的：布局视口的宽度应与理想视口的宽度一致

属性

属性 解释说明
width 宽度设置的是viewport宽度，可以设置device-width特殊值
initial-scale 初始缩放比，取值为大于0的数字
maximum-scale 最大缩放比，取值为大于0的数字
minimum-scale 最小缩放比，取值为大于0的数字
user-scalable 用户是否可以缩放，yes或no(1或0)

标准viewport设置

  1. 视口宽度和设备保持一致 ( width = device-width )
  2. 视口的默认缩放比例为1.0 ( initial-scale = 1.0 )
  3. 不允许用户自行缩放 ( user-scalable = no )
  4. 最大允许的缩放比例为1.0 ( maximum-scale = 1.0 )
  5. 最小允许的缩放比例为1.0 ( minimum-scale = 1.0 )

物理像素&物理像素比

物理像素(点)

物理像素(点)指的是屏幕显示的最小颗粒，是物理真实存在的。这是厂商在出厂时就设置的，比如iphone 6/7/8的物理像素(点)是750×1334

物理像素比

一个px能显示的物理像素点的个数，称为物理像素比或者屏幕像素比

注意

开发时的1px不一定等于一个物理像素，在PC端页面1px等于1个物理像素，但在移动端就不尽相同

发展

PC端和早期的手机屏幕是 1CSS像素 = 1 物理像素，随之Retina(视网膜屏幕)显示技术的发展，可以把更多的物理像素点压缩至一块屏幕里，从而达到更高的分辨率，并提高屏幕显示的细腻程度。

移动端技术解决方案

移动端浏览器

移动端浏览器基本以webkit内核为主，因此我们就考虑webkit兼容性问题，即浏览器的私有前缀只需添加webkit即可，并可以放心使用H5标签和CSS3样式

CSS初始化

移动端CSS初始化（CSS reset）推荐使用 normalize.css，优点如下:

  • 保护了有价值的默认值
  • 修复了浏览器的bug
  • 模块化
  • 拥有详细的文档

官网地址 : https://necolas.github.io/normalize.css/

CSS3 盒子模型

  • 传统盒子模型: 盒子的宽度 = width + border + padding
  • CSS3盒子模型: 盒子的宽度 = width
